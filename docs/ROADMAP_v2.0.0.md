## 背景

v1.7.0 已交付核心聊天、好友系统、AI 面板（DeepSeek）、中英文切换。2026-05-30 全量代码审计发现 **17 项关键差距**，分为三类：

| 类别 | 问题 | 严重度 |
|------|------|--------|
| 基础设施 | mini-asio 单线程 poll() 无法承载 500 并发 | 🔴 致命 |
| 基础设施 | 数据库 JSON 文档模型每次操作加载整张表到内存 | 🔴 致命 |
| 基础设施 | mini-spdlog / mini-json 能力不足 | 🟠 高 |
| 安全 | CSP=null、API Key 明文 base64、bcrypt 阻塞事件循环 | 🟠 高 |
| 功能 | 无消息编辑/删除/回应、文件传输 stub、群组管理无 UI | 🟠 高 |
| 质量 | 零客户端测试、无 Error Boundary、无 CI/CD gate | 🟠 高 |

## v2.0.0 之后 —— 方向性指引

以下方向建立在 v2.0.0 架构可迭代性之上，不承诺具体版本号：

| 方向 | 说明 | 契机 |
|------|------|------|
| AI 深度集成 | @AI 问答、消息翻译、智能回复建议 | `AIService` 插件接口已就绪 |
| 语音/视频通话 | LAN 环境 WebRTC P2P，信令走现有 TCP | 需新增信令协议 + 客户端 WebRTC |
| 跨网段穿透 | 跨 VLAN/VPN 服务发现 + 中继模式 | mDNS/广播 + 中继节点 |
| 管理控制台 | Web 面板：用户/群组管理、消息审计、服务监控 | 复用 SQLite + 新增 HTTP API |
| 插件生态 | 消息钩子、命令注册、UI 扩展点 | AI Provider 已是插件雏形 |

### 可持续迭代的架构保障

| 保障 | v1.7.0 现状 | v2.0.0 目标 |
|------|------------|------------|
| 协议可扩展 | switch-case 硬编码 | Handler 注册表 + definitions.json 唯一真源 |
| DB 可迁移 | migration v1 单次 | 版本递进框架，任意版本 → 最新 |
| AI 可插拔 | 2 provider 硬编码 | `AIService` 接口 + 运行时注册 |
| CI/CD | 无门禁 | 4 层 gate 脚本，无人值守运行 |
| 测试覆盖 | CTest 5 + 客户端 0 | CTest 10+ + Vitest 45+ + E2E 5+ |
| 开发文档 | CLAUDE.md 部分过时 | 准确 + 操作指南 |
